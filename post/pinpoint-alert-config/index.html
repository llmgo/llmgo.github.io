Pinpoint 告警配置

组必须是以数字构成

  1. 选择告警

  1. 添加告警

  1. 告警类型

以下规则中，请求是当前应用发送出去的，当前应用是请求的发起者。

类型 含义
SLOW COUNT / 慢请求数 发送到应用程序的慢速请求数超过配置的阈值时触发
SLOW RATE / 慢请求比例 发送到应用程序的慢速请求的百分比（％）超过配置的阈值时触发。
ERROR COUNT / 请求失败数 发送到应用程序的失败请求数超过配置的阈值时触发。
ERROR RATE / 请求失败率 发送到应用程序的失败请求的百分比（％）超过配置的阈值时触发。
TOTAL COUNT / 总数量 发送到应用程序的所有请求数超过配置的阈值时触发。

以下规则中，请求是发送给当前应用的，当前应用是请求的接收者。

类型 含义
SLOW COUNT TO CALLEE / 被调用的慢请求数量 当应用程序发送的慢速请求数超过配置的阈值时触发。您必须在“注释”字段中指定域或地址（IP，端口）：例如，www.naver.com，127.0.0.1:8080
SLOW RATE TO CALLEE / 被调用的慢请求比例 当应用程序发送的慢速请求的百分比（％）超过配置的阈值时触发。您必须在“注释”字段中指定域或地址（IP，端口）：例如，www.naver.com，127.0.0.1:8080
ERROR COUNT TO CALLEE / 被调用的请求错误数 当应用程序发送的失败请求数超过配置的阈值时触发。您必须在“注释”字段中指定域或地址（IP，端口）：例如，www.naver.com，127.0.0.1:8080
ERROR RATE TO CALLEE / 被调用的请求错误率 当应用程序发送的失败请求的百分比（％）超过配置的阈值时触发。您必须在“注释”字段中指定域或地址（IP，端口）：例如，www.naver.com，127.0.0.1:8080
TOTAL COUNT TO CALLEE / 被调用的总数量 当应用程序发送的所有请求数超过配置的阈值时触发。您必须在“注释”字段中指定域或地址（IP，端口）：例如，www.naver.com，127.0.0.1:8080

以下规则中，和请求无关，只涉及到应用和系统的状态

类型 含义
HEAP USAGE RATE / 堆内存使用率 当应用程序的堆使用率（％）超过配置的阈值时触发。
JVM CPU USAGE RATE / JVM CPU使用率 当应用程序的CPU使用率（％）超过配置的阈值时触发。
SYSTEM CPU USAGE RATE / 系统CPU使用率 当应用程序的CPU使用率（％）超过配置的阈值时，发送警报。
DATASOURCE CONNECTION USAGE RATE / 数据源连接使用率 当应用程序的数据源连接使用率（％）超过配置的阈值时触发。
DEADLOCK OCCURRENCE / 死锁 在应用程序中检测到死锁条件时发送警报。
FILE DESCRIPTOR COUNT / 文件描述符数 当打开的文件描述符的数量超过配置的阈值时，发送警报。
